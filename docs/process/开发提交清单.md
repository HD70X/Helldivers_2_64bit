# 开发提交清单（Checklist）

每次功能提交前建议检查：

## 1) 结构

- [ ] 新脚本放在正确目录（Core / GameData / GamePlay / UI）
- [ ] 没有把玩法逻辑硬塞进场景
- [ ] 文件名与类名语义一致

## 2) 联机边界

- [ ] 是否明确 authority（Player/Enemy/Projectile）
- [ ] 是否避免把网络逻辑和玩法逻辑强耦合

## 3) 数据

- [ ] 运行时结构与存档结构是否可互转
- [ ] 新增字段是否考虑默认值/兼容

## 4) 地图与资源

- [ ] 是否避免在同一 PR 混入大量无关地图改动
- [ ] 是否有临时文件、自动生成垃圾文件

## 5) 文档

- [ ] 是否更新 `docs/README.md` 索引
- [ ] 是否补充对应设计文档
