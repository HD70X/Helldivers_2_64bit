# 地图设计规范（草案）

> 这是地图专题开始前的“协作规范”，先约束流程，后填具体关卡设计。

## 1. 目标

- 降低地图文件合并冲突。
- 提高地图改动的可审查性与可回滚性。

## 2. 分块规则（建议）

- 采用 chunk 子场景组织地图，而不是单个超大场景。
- 每个 chunk 建议只包含：
  - TerrainBase
  - Details
  - 交互触发器（可选）
- 命名建议：`chunk_<x>_<y>.tscn`

## 3. 图层约定

- Sky：纯视觉，不参与碰撞。
- TerrainBase：地形与碰撞主层。
- Details：装饰层，默认不承载玩法逻辑。
- 玩法触发物请使用独立 Node，不直接混入 TileMap 数据。

## 4. 协作约定

- 同一时间尽量单人改同一 chunk。
- 大改地图前先开设计文档与任务卡。
- PR 描述中必须写：改动 chunk 列表、碰撞改动点、出生点/路径点影响。

## 5. 联机注意事项

- 地图碰撞必须以主机为准（权威碰撞）。
- 客机可做显示层插值，但命中与位移判定应可回放校正。

## 6. 验收清单

- 出生点可达性
- 路径连通性
- 地形边缘卡点检查
- 低帧率下移动稳定性
