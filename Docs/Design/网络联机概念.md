## 网络模型

- 游戏采用玩家主机（peer-hosted）的合作联机架构。
- 匹配由一个轻量级的外部服务负责。
- 当对局开始后，其中一名玩家会成为主机（Host）。
- 主机对战斗关键的模拟拥有最终权威：
	- 敌人 AI
	- 伤害
	- 死亡
	- 掉落
	- 任务进度
- 客户端仅对本地输入意图与即时表现负责：
	- 本地移动预测
	- 瞄准方向
	- 换弹动画
	- 切换武器动画
- 最终的玩法结果由主机校验并广播给所有玩家。

## 联机概览

游戏采用玩家主机式的合作联机模型。

使用轻量级的匹配服务来连接玩家。

对局开始后，其中一名玩家成为主机并运行权威模拟。

客户端将输入发送给主机，并在本地预测玩家的即时动作。

主机负责校验并同步战斗结果。

### 权威归属

本地移动 → 客户端预测
瞄准方向 → 客户端
换弹 → 客户端预测，主机确认
敌人 AI → 主机
伤害 → 主机
敌人死亡 → 主机
任务进度 → 主机

## Network Model

- The game uses a peer-hosted coop architecture.
- Matchmaking is handled by a lightweight external service.
- Once a match starts, one player becomes the Host.
- The Host is authoritative for combat-critical simulation:
	- enemy AI
	- damage
	- death
	- drops
	- mission progress
- Clients are authoritative only for local input intent and immediate presentation:
	- local movement prediction
	- aim direction
	- reload animation
	- weapon switch animation
- Final gameplay results are validated and broadcast by the Host.

## Networking Overview

The game uses a peer-hosted cooperative networking model.

A lightweight matchmaking service is used to connect players.

Once a match begins, one player becomes the Host and runs the authoritative simulation.

Clients send input to the Host and locally predict immediate player actions.

The Host is responsible for validating and synchronizing combat results.

### System Authority

Local Movement   → Client prediction
Aim Direction    → Client
Reload           → Client predicted, Host confirmed
Enemy AI         → Host
Damage           → Host
Enemy Death      → Host
Mission Progress → Host
