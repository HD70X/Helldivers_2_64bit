# 短期 Demo 规划（内部备份）

> 仅作内部规划记录。
> 不在仓库最外层 README 暴露，不作为对外公开说明。

## 项目定位

这是一个致敬 **Helldivers 2** 的粉丝项目。

- 引擎：Godot
- 类型：2D 像素风平台动作射击游戏
- 风格参考：合金弹头式高精度像素表现

## 短期目标

先做出一个**基本可玩的 Demo**，用于在社区中展示项目可行性，并吸引更多关注、帮助与支持。

这一步的目标不是完成复杂的大工程，而是尽快拿出一个“看起来已经能玩、而且有潜力”的版本。

## Demo 的核心诉求

为了让外部观众、潜在协作者、可能的美术/音乐支持者快速理解项目价值，Demo 至少应包含：

1. **大厅场景**
   - 作为进入游戏后的基础展示空间
   - 承载开始游戏、进入教学、后续扩展入口等功能

2. **教学场景**
   - 用于展示基础玩法
   - 让试玩者快速理解角色操作、武器使用和场景交互

3. **几乎完整的角色操控表现**
   - 角色移动手感
   - 动画表现
   - 射击反馈
   - 基础交互/演出感

## 为什么优先这些

如果 Demo 不能体现：

- 角色操作是否扎实
- 场景是否可进入、可游玩
- 视觉与动作反馈是否有吸引力

那么它就很难在社区中引起兴趣，也不容易吸引真正愿意提供帮助的人加入。

## 对外展示预期

一个成功的短期 Demo 应该让人产生这些感觉：

- “这项目已经不是概念图了，是真的能玩。”
- “基础动作和射击体验已经有雏形。”
- “如果补上更多内容、美术、音乐，会很有潜力。”

## 协作期望

短期内如果能吸引到外部帮助，最有价值的方向包括：

- 美术支持
- 音乐 / 音效支持
- 后续玩法与工程协作支持

其中，**美术或音乐支持都会是非常大的进步**。

## 当前策略判断

在短期内，优先级应倾向：

- 先让 Demo 看起来可行、可玩、可展示
- 再考虑更完整的结构升级与复杂工程
- 复杂系统可以预留方向，但暂不抢占当前 Demo 的开发重心
