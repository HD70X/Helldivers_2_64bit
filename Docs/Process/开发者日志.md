# 开发提交清单（Checklist）

每次功能提交前建议检查：

## 1) 结构

- [ ] 新脚本放在正确目录（Core / GameData / GamePlay / UI）
- [ ] 没有把玩法逻辑硬塞进场景
- [ ] 文件名与类名语义一致

## 2) 联机边界

- [ ] 是否明确 authority（Player/Enemy/Projectile）
- [ ] 是否避免把网络逻辑和玩法逻辑强耦合

## 3) 数据

- [ ] 运行时结构与存档结构是否可互转
- [ ] 新增字段是否考虑默认值/兼容

## 4) 地图与资源

- [ ] 是否避免在同一 PR 混入大量无关地图改动
- [ ] 是否有临时文件、自动生成垃圾文件

## 5) 文档

- [ ] 是否更新 `docs/README.md` 索引
- [ ] 是否补充对应设计文档

## 2026-03-10 Demo 入口闭环整理

### 本次目标

- 为当前项目建立最小 Demo 入口闭环
- 不处理飞船大厅、完整训练场、角色大重构
- InputMap 由开发者手动维护

### 本次完成

- 将项目启动入口切换为主菜单场景 `res://Scene/UI/main_menu.tscn`
- 接通主菜单按钮逻辑：
  - `StartButton` 进入 `res://Scene/TestMap.tscn`
  - `ExitButton` 退出游戏
- 保持 `SettingButton` 为禁用占位，不展开设置系统
- 为 `TestMap` 增加最小场景控制脚本，允许通过 `ui_cancel` 返回主菜单
- 收口 `character.gd` 中的输入动作命名：
  - 开火动作统一为 `fire`
  - 换弹动作统一为 `reload`
  - 左右移动动作通过常量统一为 `move_left` / `move_right`

### 涉及文件

- `project.godot`
- `Scene/UI/main_menu.tscn`
- `Scene/TestMap.tscn`
- `Script/MainMenu/main_menu.gd`
- `Script/GamePlay/demo_scene_controller.gd`
- `Script/GamePlay/character.gd`

### 当前结果

- 游戏启动后应先进入主菜单
- 点击开始后应进入 `TestMap`
- 在 `TestMap` 中按 `ui_cancel` 应返回主菜单
- 当前菜单闭环已具备最小可验证结构

### 备注

- 本次改动优先复用现有场景和脚本框架，避免扩大范围
- 后续可在该基础上继续补 Demo 提示、训练引导和更正式的入口包装
